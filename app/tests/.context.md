# Context: app/tests/

## Purpose & Responsibility
Contains the automated test suite for the application. Uses `pytest` and `fastapi.testclient`. The testing strategy emphasizes Test-Driven Development (TDD) and covers security, API correctness, and frontend integration.

## Key Test Areas
- **Security (`test_security.py`):** Validates path resolution, jail isolation, and fail-fast configuration checks.
- **API (`test_api.py`, `test_mobile_api.py`):** Tests both web-facing and mobile-optimized JSON endpoints, including CRUD and search operations.
- **Frontend Integration (`test_artplayer_integration.py`):** Ensures that templates correctly include assets and implement client-side logic (e.g., Artplayer, prefetching).
- **Authentication (`test_auth_routes.py`, `test_mobile_auth.py`):** Verifies JWT, session, and basic auth flows.
- **Infrastructure (`test_supervisor_interpreter.py`, `test_user_sync.py`):** Tests the orchestration and synchronization logic.

## Dependencies
- `pytest`: Test runner.
- `httpx`: For `TestClient` requests.
- `pytest-cov`: For coverage reporting.
- `unittest.mock`: Extensively used for mocking the Copyparty backend and filesystem.
