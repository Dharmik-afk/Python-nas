# Context: app/backend/routes/

## Purpose & Responsibility
Defines the API endpoints for the application. Organized by functional area (API, Auth, Download, Upload).

## Key Symbols
- `app/backend/routes/api_routes.py`:
    - `/fs/list`: Mobile-optimized JSON directory listing.
    - `/fs/search`: Search proxy (JSON and HTML partials).
    - `/fs/rename`, `/fs/mkdir`, `/fs/{full_path}` (DELETE): Filesystem manipulation endpoints.
- `app/backend/routes/auth_routes.py`:
    - `/token`: OAuth2 compatible JWT login for mobile/API clients. Performs "handshake" verification with Copyparty.
    - `/login`: Form-based login for web browsers.
- `app/backend/routes/download_routes.py`:
    - `/download/{path}`: The primary proxy for file streaming, thumbnail generation, and Range request handling.
- `app/backend/routes/upload_routes.py`:
    - `/upload/{path}`: Proxies file uploads to the backend.

## Dependencies
- `fastapi`: API routing and dependency injection.
- `app.core.auth`: For `auth_required` dependency.
- `app.backend.services.copyparty_service`: Core logic for file operations.
